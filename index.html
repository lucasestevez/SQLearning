Banco de Dados

Segundo Korth, Silberschatz e Sudarshan, um banco de dados “[...] é uma coleção de dados inter-relacionados, representando informações sobre um domínio específico”.

Um exemplo típico de onde um banco de dados pode ser utilizado é em um sistema de gestão de uma biblioteca.

Aqui está uma situação simplificada:

Imagine que esteja desenvolvendo um sistema para uma biblioteca, no qual os utilizadores possam pesquisar e requisitar livros. Um banco de dados seria útil para guardar informações como:

Detalhes do Livro: Título, autor, número de páginas, género, data de publicação, ISBN, etc.

Detalhes do Utilizador: Nome, morada, número de identificação, histórico de requisições, etc.

Requisições Atuais: Quais livros foram requisitados por quem e quando devem ser devolvidos.

Stock de Livros: Quantas cópias de cada livro a biblioteca possui.

Reservas: Se algum livro foi reservado por um utilizador e quando estará disponível para requisição

Um banco de dados pode facilitar o gerenciamento dessas informações de forma organizada e eficiente. Por exemplo, quando um usuário faz uma pesquisa por um livro, o sistema consulta o banco de dados para encontrar os livros correspondentes e exibi-los. Quando um usuário faz um empréstimo, o sistema verifica se o livro está disponível no estoque, atualiza o registro de empréstimos e ajusta o estoque disponível. Quando um usuário devolve um livro, o sistema atualiza o registro de empréstimos e o estoque disponível. E assim por diante.

Portanto, um banco de dados é essencial para manter a integridade e a consistência das informações em um sistema de gerenciamento de bibliotecas, garantindo que os dados estejam disponíveis e acessíveis quando necessário.

Como administrar um Banco de Dados

A questão da administração de um banco de dados é ampla e complexa, sendo debatida por diversos autores. Em resumo, você deve ter conhecimento de que tipo de informação está alocada nas tabelas do seu banco de dados e, principalmente, como manipular as informações que ali estão inseridas da forma que necessita.

Um sistema de banco de dados é formado pelo software de banco de dados, o repositório de tabelas denominado database e as próprias tabelas em si ou tables. Dessa forma, um banco de dados para um sistema de agenda telefônica teria o serviço do banco de dados executado, uma database e, dentro desta, estariam as tabelas para gravar as informações de nome, endereço, contato telefônico, e-mail, aniversário, foto, etc.

Portanto, para você conseguir realizar uma boa administração do banco de dados, é necessário conhecer o negócio ou o projeto ao qual ele servirá. De nada adianta ter tabelas que não são necessárias, pois isso impacta na performance do serviço. No exemplo da agenda telefônica, se não existisse o campo “contato telefônico”, o sistema ficaria inútil, pois uma agenda tele- fônica precisa, obrigatoriamente, conter o campo para cadastro do telefone do contato.

Como passo fundamental na administração correta de um banco de dados, é essencial que você tenha a documentação base do sistema que utilizará do banco de dados sob sua administração. Esse documento contemplará a modelagem do banco de dados, os relacionamentos entre tabelas, como será gravada cada informação, quais campos são obrigatórios e quais são opcionais, a necessidade de indexação para melhorar a velocidade de consulta, etc.

Chave Primária e Chave Estrangeira

Chave Primária: é um campo que é definido como único e não pode repetir-se naquela tabela.

Chave estrangeira: ocorre quando pegamos um atributo de outra tabela que é chave primária nela.

Tipos de Atributos

Existem 5 tipos de atributos mais comumente usados nos Bancos de dados atualmente. São eles:

  • INT ou INTEGER: recebe apenas números inteiros. Exemplo: 1893124;
  • VARCHAR: recebe apenas caracteres alfanuméricos. Exemplo: Paulo Algusto Silva;
  • DATE: recebe apenas datas válidas. Exemplo: 25/09/2035;
  • TIME: recebe apenas valores expressos em hora. Exemplo: 13:06;
  • BOOL ou BOOLEAN: recebe apenas número 0 ou 1. Serve para indicar se algo é verdadeiro ou falso, positivo ou negativo, etc. Exemplo: 0 (o sistema sabe que, se esse campo for 0, o cliente está verdadeiro para comprar a prazo).
Sistema de Gerenciamento de Banco de Dados (SGBD)

Sistemas de Gerenciamento de Banco de Dados (inglês Data Base Management System), são um conjunto de software usados para gerenciar uma base de dados. De forma simples, um banco de dados armazena informações e para estruturar esse banco de dados, de forma que todas as informações estajam no lugar certo na hora de receber elas, utilizamos os SGBDs.

Nos dias atuais, temos 4 SGBDs que dominam o cenário e um 5º está em crescimento. São eles:

  • MySQL;
  • MariaDB;
  • Microsoft SQLServer;
  • PostgreSQL;
  • Oracle;
  • MongoDB (em expansão).
